# 과제1: VanillaJS로 Todo 앱 만들기
## 폴더구조
```
todo-vanilla/
├── index.html
├── style.css
├── app.js
└── README.md
```

## 사용 프롬프트 정리
### 0. 전체 구조 잡기
```markdown
나는 Vanilla JS로 Todo 웹 앱을 만들 거야.
아래 조건을 기억하고, 이후 내가 기능 구현을 요청할 때 이 조건에 맞게 코드를 작성해줘.

[조건]
- HTML / CSS / Vanilla JS만 사용해 (라이브러리, 프레임워크 사용 금지)
- 파일은 index.html, style.css, app.js 세 개로 구성해
- 깔끔하고 미니멀한 생산성 앱 스타일로 메인 컬러는 #672be0을 사용해
- 변수명과 함수명은 역할을 알 수 있도록 명확하게 작성해
- 코드에는 동작 방식을 이해할 수 있도록 주석을 달아줘
- 기능을 추가하거나 수정할 때는 항상 index.html, style.css, app.js 전체 코드를 파일별로 나눠서 보여줘
- 모든 작업을 마친 후에는 어떤 기능을 구현했는지 체크리스트로 답변해

위 조건을 이해했으면 '준비됐어'라고만 답해줘.
```

### 1. Todo CRUD 구현
```markdown
Todo 앱의 HTML 레이아웃과 기본 CRUD 기능을 만들어줘.

- 텍스트 입력창과 추가 버튼으로 Todo를 생성할 수 있어야 해
- 생성된 Todo는 목록으로 표시되고, 각 항목마다 수정 / 완료 / 삭제 버튼이 있어야 해
- 입력값이 비어있으면 Todo가 생성되지 않고 안내 메시지를 표시해줘
- 완료된 Todo는 텍스트에 취소선이 표시되어야 해
```

### 2. 상태별 필터링 구현
```markdown
기존 코드에 상태별 필터링 기능을 추가해줘.

- 전체 / 진행 중 / 완료 탭을 클릭하면 해당 상태의 Todo만 표시돼야 해
- 현재 선택된 탭은 시각적으로 구분되는 스타일이 적용되어야 해
```

### 3. Todo 일간 뷰 구현
```markdown
기존 코드에 일간 뷰 기능을 추가해줘.

- 화면 상단에 오늘 날짜를 표시하고, 이전 / 다음 버튼으로 날짜를 이동할 수 있어야 해
- Todo를 생성할 때 현재 선택된 날짜를 함께 저장해야 해
- 선택된 날짜에 해당하는 Todo만 목록에 표시돼야 해
```

### 4. 로컬스토리지 연동
```markdown
기존 코드에 로컬스토리지 연동 기능을 추가해줘.

- Todo를 추가 / 수정 / 삭제 / 완료 처리할 때마다 로컬스토리지에 저장되어야 해
- 페이지를 새로고침하면 로컬스토리지에서 데이터를 불러와 목록을 복원해야 해
- 데이터는 JSON.stringify / JSON.parse를 사용해서 저장하고 불러와줘
```

### 5. Todo 주간 뷰 구현

기존 코드에 주간 뷰 기능을 추가해줘.
```markdown
- 이번 주 월요일부터 일요일까지의 날짜를 가로로 나열해줘
- 이전 주차, 다음 주차로 넘길 수 있는 버튼을 만들어줘
- 날짜를 클릭하면 해당 날짜의 Todo만 목록에 표시돼야 해
- 각 날짜 아래에 해당 날짜의 Todo 개수를 표시해줘
- 오늘 날짜는 시각적으로 구분되는 스타일을 적용해줘
```
